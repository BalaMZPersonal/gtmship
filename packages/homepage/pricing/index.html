Before you look at pricing

Self-hosting is still the main path

GTMship is built as an open-source, deploy-to-your-cloud product. The paid options here are intentionally optional and designed for teams that want less infra work or want GTM workflows built and operated for them.

Why the hosted entry price can stay low

Modern scale-to-zero infrastructure stays inexpensive for light-to-moderate workflow volume. That keeps the hosted entry tier practical without turning GTMship into a task-metered SaaS.

Local-only run

You do not need to deploy on day one. GTMship supports a local-first workflow where you connect systems, generate the automation, validate it, and preview runs before you ever push it to a cloud runtime.

Start local

Build and test on your machine

Use the local dashboard and auth flow to connect tools, iterate in Workflow Studio, and prove the workflow against sample input while everything is still under one operator's control.

Deploy later

Move to cloud only when the workflow needs to stay on

Cloud deployment becomes useful when you need stable webhooks, schedules, shared team access, or a runtime that should keep running after your local session ends.

Docs path
